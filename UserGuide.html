MyProject User Guide:

Introduction:

MyProject (MP) is for team leaders who prefer to use a desktop app for managing their projects. MP is optimized for those who prefer to work with a Command Line Interface (CLI) while still having the benefits of a Graphical User Interface (GUI). With MP it is a one-stop app to store and manage your projects and their relevant information, this is especially useful if you are always managing multiple projects across different teams. Interested? Jump to the Section 2, “QuickStart” to get started. Enjoy!

Quick Start:

  1. Ensure you have Java 11 or above installed in your Computer.

  2. Download the latest myproject.jar here.

  3. Copy the file to the folder you want to use as the home folder for your Address Book.

  4. Type the command in the command box and press Enter to execute it. e.g. typing help and pressing Enter will open the help window.

  5. Some example commands you can try:

    1. list : lists all contacts

    2. exit : exits the app

  6. Refer to Section 3, “Features” for details of each command.

Features

  1. Help

    1. Viewing help: help

      1. Shows a list of commands.

      2. Format: help

  2. List

    1. List Projects: listProjects

      1. Shows the list of projects.

      2. Format: listProjects

    2. List Members: listMembers

      1. Shows the list of members involved with the specified project. If no project is specified, a list of all projects with its corresponding members is shown.

      2. Format: listMembers [pr/PROJECT_NAME]

  3. Add

    1. Add Project: addProject

      1. Adds a project to your list of projects

      2. Format: addProject pr/PROJECT_NAME [d/PROJECT_DESCRIPTION] [dl/DEADLINE] [m/MEMBERS] [ta/TASKS] [t/TAG]…​

      3. Example: addProject pr/GET1031 project dl/October 11 Tuesday 2359 m/John Lim_Mark Tan_David Chan ta/1. Come up with project outline 2.Research on project topic t/NUS

    2. Add Meeting: addMeeting

      1. Adds a meeting to the list of meetings of your current project.

      2. Format: addMeeting p/PROJECT_NAME [dt/tentative days for a meeting] [m/MEMBERS INVOLVED] [a/AGENDA]…​

      3. Example: addMeeting p/GET1031 project dl/October 11 - October 17 m/John Lim_Mark Tan_David Chan a/decide on the structure of presentation

    3. Add Member: addMember

      1. Adds a person to addressBook. After a person is added, they can then be linked to a project by editing the project.

      2. Format: addMember n/NAME p/PHONE_NUMBER e/EMAIL a/ADDRESS [t/TAG]…​

      3. Example: addMember n/John Doe p/98765432 e/johnd@example.com a/John street, block 123, #01-01

    4. Add Task: addTask

      1. Adds an unchecked task to the list of tasks of your current project.

      2. Format: addTask s/DESCRIPTION c/TIME

      3. Example: addTask s/Finish GUI c/06/10/2019 1600

  4. Edit

    1. Edit project: edit

      1. Edit the attributes of the project.

      2. Format: edit INDEX [s/DESCRIPTION] [n/NAME]

      3. Examples: __edit 5 pr/OP1 pd/Oral Presentation on Negotiation Techniques dl/25/09/2019 2359

    2. Edit task: editTask

      1. Edits task with the specified index of the task list of current project. To check off task, input d/(i.e. editTask 1 d/). Tasks are automatically unchecked if d/ is not in the input.

      2. Format: editTask INDEX [s/DESCRIPTION] [c/TIME] [d/]

      3. Example: editTask 1 c/05/05/2019 d/

  5. Delete

    1. Delete person: delete

      1. Deletes specified person from the addressbook.

      2. Format: delete INDEX

      3. Example: delete 1

    2. Delete task: deleteTask

      1. Deletes task of specified index from the task list of the current project.

      2. Format: deleteTask INDEX

      3. Example: delete 1

  6. Meeting

    1. Store meeting minutes: attach

      1. Upload text file that contains the meeting minute for a specific meeting.

      2. Format: attach [mt/MEETING_INDEX] [dl/DEADLINE] [ta/TASKS] [t/TAG]…​

    2. Generate next meeting date: generate

      1. Come up with next meeting date.

      2. Format: generateTiming [mt/MEETING_INDEX]

    3. Send reminder: Reminder

      1. Sends a reminder to all members of the current project.

      2. Format: sendReminder

    4. Adds meeting to the Project: addProjectMeeting

      1. Adds meeting to a currently checked out project.

      2. Format: addProjectMeeting [c/ dd/MM/yyyy HHmm] [s/MEETING_DESCRIPTION]

  7. Sort

All sort methods follow this set of index/order pairing. Some indexes are specific to certain methods(such as 3 and 4 are specific to sortTask). Refer to each individual method to find out which indexes are available.

1 — Alphabetical order

2 — Increasing order of time

3 — Whether tasks are done.

4 — Whether tasks are done and then by increasing order of time.

  1. Sort tasks: sortTask

    1. Sorts tasks in the task list of current project based on given index.

    2. Format: sortTask INDEX(Integer value from 1-4)

    3. Example: sortTask 1 (Sort tasks by alphabetical order)

  2. Sign in to Account: signIn

    1. Signs in to the Account that that user has.

    2. Format: signIn [ac/ACCOUNT_EMAIL_ADDRESS] [pa/PASSWORD]

  3. Signs out from user’s account: logOut

    1. Signs out from currently signed in account.

    2. Format: logout

  4. Sends personal Email: sendMail

    1. Sends an email to one person in the AddressBook.

    2. Format: sendMail [r/RECIPIENT_OF_EMAIL] [su/SUBJECT] [me/MESSAGE_BODY]

  5. Sends broadcast Email: broadcastMail

    1. Sends an email to all members in the Project.

    2. Format: broadcastMail [su/SUBJECT] [me/MESSAGE_BODY]
